页面导航

@layer 是现代浏览器的新特性，主要用来对 CSS 逻辑进行分层管理。

在我们之前的理解，CSS 的优先级，由低到高默认如下：

  1. CSS 书写顺序，底部优先，覆盖顶部样式语句。
  2. CSS 特异性
  3. 行内样式
  4. !important 语句

而 @layer 语法则是在特异性的上层进行控制，优先级高于特异性：

  1. CSS 书写顺序，底部优先，覆盖顶部样式语句。
  2. CSS 特异性
  3. @layer
  4. 行内样式
  5. !important 语句

  4. 圆角与边框 (Borders & Radius)
  5. 阴影与特效 (Shadows & Effects)

  7. 动画与过渡 (Animations & Transitions)

1-2.自定义主题 ​

我们可以定向的覆盖原有主题或者新增自定义主题：

css
@theme {
	/* 覆盖 */
	--spacing: 1rem;
	/* 新增 */
	--color-blue: blue;
}

然后我们就可以直接使用 mt-2 或者 bg-blue 等工具类。

1-3.自定义变量主题 ​

如果我们想要定义的 theme 从全局变量中获取，那么我们可以使用 @theme inline 定义：

css
:root {
	--color-blue: blue;
}

@theme inline {
	--color-blue: var(--color-blue);
}

2.Layer base ​

在日常开发中，我们一般会使用 normalize.css 来重置默认的浏览器样式。

而在 Tailwind CSS 中，默认提供了 @layer base 模块，它会在 @import 'tailwindcss' 中默认引入。

当然我们也可以使用 @layer base 来额外定义 base 样式：

css
@layer base {
  h1 {
    font-size: var(--text-2xl);
  }
  h2 {
    font-size: var(--text-xl);
  }
}

3.Layer components ​

如果在实际业务中，部分 components 复用多次，那么可以考虑封装成公共 class：

css
@layer components {
  .card {
    background-color: var(--color-white);
    border-radius: var(--rounded-lg);
    padding: var(--spacing-6);
    box-shadow: var(--shadow-xl);
		@apply text-center;
  }
}

4.Layer utilities ​

Tailwind 内置了一些 utilities，但是 Tailwind 也允许自定义 utilities。

当内置 utilities 不满足需求，或者 CSS 出现了兼容问题时，我们可以创建自己的 utilities。

css
@utility content-auto {
  content-visibility: auto;
}

@utility scrollbar-hidden {
  &::-webkit-scrollbar {
    display: none;
  }
}

@utility tab-* {
  tab-size: --value(--tab-size-*);
}
