## 含义
`AGENTS.md` 是项目级的 Agent 操作手册，用来明确“允许做什么、如何做、用什么工具做”。它能把团队约定固化下来，让 Agent 在仓库内工作更稳定、更安全。

## 作用
- 统一行为：减少风格与流程差异。
- 降低沟通成本：把默认假设写成规则。
- 控制风险：限制高风险操作，减少误操作。

## 推荐结构
- 目标与范围：允许/不允许的任务边界。
- 环境与依赖：安装方式、运行命令、版本要求。
- 代码与内容规范：命名、目录结构、格式化规范。
- 任务流程：常见任务的步骤与验收标准。
- 安全与审查：高风险操作需确认或走审查流程。

## 示例（AGENTS.md 模板）
```md
# AGENTS.md

## 范围
- 允许：修复 bug、优化文档、添加小功能。
- 不允许：重构核心架构、删除历史数据、改动生产配置。

## 环境
- 包管理：pnpm。
- 启动：`pnpm dev`。
- 构建：`pnpm build`。

## 规范
- 优先编辑 `docs/` 下的内容。
- 新增文档必须包含标题与清晰的小节结构。
- 改动需说明影响范围与验证方式。

## 安全
- 涉及删除文件或大量改动时先确认。
- 不执行破坏性命令（如 `git reset --hard`）。
```

## 适用于本仓库的示例
下面是一个更贴合本仓库（VitePress）的示例内容，可作为真实 `AGENTS.md` 的起点：
```md
# AGENTS.md

## 范围
- 允许：完善文档、更新示例、调整 VitePress 配置。
- 不允许：重构站点架构、删除大量页面、修改 CI 规则。

## 环境
- 包管理：`pnpm`（见 `package.json` 的 `packageManager` 字段）。
- 本地开发：`pnpm dev`。
- 构建：`pnpm build`。
- 预览：`pnpm preview`。
- 说明：`pnpm test` 当前未配置，不要作为默认步骤。

## 内容规范
- 文档优先放在 `docs/` 目录。
- 文档需包含标题与清晰的小节结构。
- 链接需可访问，避免断链。

## 工作流程
- 修改前先定位相关文件与上下文。
- 修改后说明变更点与影响范围。
- 如无测试，说明未运行测试原因。

## 安全
- 删除/迁移文件需先确认。
- 避免破坏性命令（如 `git reset --hard`）。
```
