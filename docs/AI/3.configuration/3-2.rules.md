## 含义
Rules 用于明确 Agent 在执行任务时必须遵守的规则与约束，覆盖行为、质量、安全与协作流程。它相当于“执行边界与质量底线”。

## 作用
- 约束高风险操作。
- 统一质量标准与输出格式。
- 减少返工与误解。

## 常见规则类型
- **行为规则**：先确认再执行、遇到不确定先提问。
- **质量规则**：代码需可读、文档需结构化。
- **安全规则**：禁止破坏性命令、限制敏感路径。
- **协作规则**：改动需说明、测试结果需报告。

## 示例（Rules 模板）
```md
# Rules

## 行为
- 不确定时先提问，不自行假设。
- 修改范围超出需求时必须确认。

## 质量
- 文档必须有标题和清晰的小节。
- 代码变更需说明影响范围。

## 安全
- 禁止执行 `git reset --hard`、`rm -rf /` 等破坏性命令。
- 删除文件前必须确认。

## 协作
- 提交前说明变更点与未完成事项。
- 若未运行测试，必须说明原因。
```

## 使用建议
- Rules 应与 Config、AGENTS.md 一致。
- 规则越少越清晰，避免重复与冲突。
