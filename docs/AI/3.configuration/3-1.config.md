title: 3-1.Config
---

## 含义
Config 用来描述 Agent 在某个环境中的“运行参数与能力开关”。它通常包含模型、工具、权限、工作目录、上下文载入规则等设置，决定 Agent 能做什么、如何做。

## 作用
- 固定默认行为，减少每次手工配置。
- 控制权限与安全边界。
- 保证不同环境下的可重复性。

## 常见配置项
- **模型与推理**：默认模型、温度、最大输出长度等。
- **工具与能力**：是否启用特定工具（如 Web、MCP、文件读写）。
- **权限与沙箱**：读写目录白名单、执行命令限制。
- **工作区**：默认 `cwd`、工作路径映射。
- **上下文策略**：自动读取哪些文件、最大上下文大小。

## 配置层级（建议）
- 全局配置：个人级默认设置。
- 项目配置：仓库级覆盖设置。
- 会话配置：临时覆盖设置。

## 示例（通用模板）
```md
# config (示意)
- model: gpt-4.1
- tools: [filesystem, mcp]
- permissions:
  - read: [./docs, ./src]
  - write: [./docs]
- cwd: /path/to/project
- context:
  - include: [README.md, AGENTS.md]
  - max_tokens: 8000
```

## 使用建议
- 在团队内约定一套最小必要配置。
- 变更前先确认对安全与流程的影响。
- 与 `AGENTS.md`、`Rules` 保持一致，避免冲突。
